My work and skills

I'm a front-end developer at heart interested in creating good UI/UX for the web with pixel perfection shooting through the roof. I love structure and re-usability and believe the code should look as good as the site itself. I have worked full-stack a lot and as long as I have a Mac with an open terminal window I'm good to go.

There is no doubt JavaScript is my favorite programming language. I've developed applications for the back-end with NodeJS but mostly for the browser, the front-end. Of course Vanilla JS but also frameworks and libraries such as Angular, Vue and React. I also run across legacy code with AngularJS and jQuery. On the opposite side I love tinkering with TypeScript, module bundlers, package managers and task runners, just to mention a few. I could probably keep listing for hours.
Listing HTML as a skill might be redundant information because all web developers know HTML, right? I disagree because I’ve seen many cases where the HTML is a mess with no clear structure or coherence. This includes semantic elements such as articles, asides and sections but also a lack of accessibility in the code. Areas where SEO could be boosted and improving readability for all visitors.
When I said that I’ve seen a lot of cases where the HTML is a mess, don’t get me started on the CSS. This is an area where I have a lot of knowledge and my love for pixel perfection makes the styling of all web sites awesome. Of course I also write the styling in Less or Sass/SCSS instead of normal CSS if applicable. If there is no style guide present let’s create one... or why not accompanied with a component library to make it grow into a complete design system later on.
Let’s put all released versions into their own zip folder and place them on an external drive shared across all project members... or not. Call me crazy but I really like typing git commands in the terminal. Force push to master and reverting commits on origin. Projects with good guidelines for using version control with feature branches and pull requests makes our lives so much better. I’m using git for a lot more than just versioning web projects. For example I use it to share dotfiles and preferences across multiple computers. A scheduled script makes it all automated and my settings are always up-to-date wherever I work.
My biggest achievement in life was when I learned how to quit Vim. I think working on Linux servers is a lot of fun (Windows is the opposite). Everything is often so light weight with high stability and you always have control when writing terminal commands. This includes setting up web servers such as Apache and Nginx with SSL certificates, configure firewalls, set up Cron jobs or my favorite: writing cool bash scripts.
I’m not always deep down in an SSH session writing rad commands like “ls -la” all day, I also like designing. I started using Photoshop many years ago but today with all the different devices and screens I use vector based programs, such as Sketch and Illustrator, instead. That works much better for all the responsive designs in different resolutions I code and design for. Unfortunately I can spend way to many hours finding the best shade of blue, my favorite color by the way.
